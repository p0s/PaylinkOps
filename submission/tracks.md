# Tracks

Primary track:

- MoonPay CLI Agents

Optional secondary track:

- OpenWallet Standard

Notes:

- Primary fit is MoonPay CLI Agents because PaylinkOps uses the local `mp` binary as the live execution rail.
- The official MoonPay track asks for agents where MoonPay CLI is the primary action layer and the result is more than a basic demo.
- PaylinkOps fits that directly: it uses MoonPay CLI for real wallet and deposit actions, then adds receipts, reconciliation, and operator review on top.
- OpenWallet Standard should only be claimed if you want to argue the wallet layer as more than roadmap.
